## Token Saving Rule

Untuk menghemat token:

1. Jangan membaca semua dokumen di folder `Docs/` untuk setiap task.
2. Selalu baca `Docs/00_DOCS_INDEX.md` terlebih dahulu.
3. Setelah itu baca hanya dokumen dan section yang relevan dengan task.
4. Maksimal baca 3–4 dokumen utama per task, kecuali user meminta audit seluruh project.
5. Jika task berkaitan dengan database, prioritaskan ERD.
6. Jika task berkaitan dengan fitur, prioritaskan FRD dan Acceptance Criteria.
7. Jika task berkaitan dengan tampilan, prioritaskan UI Spec.
8. Jika task berkaitan dengan testing, prioritaskan Acceptance Criteria dan Test Plan.
9. Jika task berkaitan dengan deployment, prioritaskan Deployment & Ops.
10. Jangan membuat fitur di luar scope MVP tanpa instruksi eksplisit.
