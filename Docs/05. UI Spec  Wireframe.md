# 05. UI Spec / Wireframe

# Sistem E-Learning Training Karyawan

------

## 5.1 Ringkasan Dokumen

Dokumen UI Spec / Wireframe ini menjelaskan rancangan tampilan antarmuka Sistem E-Learning Training Karyawan.

Dokumen ini digunakan sebagai acuan untuk:

1. Menentukan halaman apa saja yang perlu dibuat.
2. Menentukan isi dan komponen pada setiap halaman.
3. Menentukan pola layout Admin dan Karyawan.
4. Menentukan reusable component agar tampilan konsisten.
5. Menentukan empty state, loading state, dan error state.
6. Menentukan warna token dan design foundation.
7. Menentukan pola wireframe untuk desktop, tablet, dan mobile.
8. Menjadi acuan implementasi Blade, Tailwind CSS, Alpine.js, dan JavaScript.

Sistem ini menggunakan gaya desain yang bersih, modern, enterprise, dan responsive. Tampilan utama menggunakan warna putih sebagai canvas, biru sebagai warna aksi utama, teks hitam/ink untuk konten utama, serta warna semantic untuk status sistem.

------

## 5.2 Prinsip Desain UI

Prinsip desain yang digunakan:

1. **Clean Enterprise Layout**
   Tampilan dibuat bersih, rapi, dan mudah dibaca untuk kebutuhan dashboard perusahaan.
2. **Reusable Component**
   Komponen seperti button, input, table, badge, modal, card, tab, dan alert harus dibuat reusable menggunakan Blade Component.
3. **Consistent Design System**
   Warna, typography, spacing, radius, dan state komponen harus konsisten di seluruh halaman.
4. **Responsive First**
   Sistem harus nyaman digunakan di laptop, tablet, dan HP.
5. **Admin Table-Based**
   Halaman Admin menggunakan table list untuk memudahkan pengelolaan data.
6. **Karyawan Card/List-Based**
   Halaman Karyawan menggunakan card/list agar lebih nyaman digunakan pada HP.
7. **Clear State Feedback**
   Sistem wajib memiliki empty state, loading state, error state, success state, dan confirmation state.
8. **No Browser Default Modal**
   Sistem tidak boleh menggunakan modal bawaan browser seperti `alert()`, `confirm()`, atau `prompt()`. Semua konfirmasi harus menggunakan custom modal.

## 5.5.7 Modal

### Modal Wajib

1. Delete permanen.
2. Nonaktifkan.
3. Aktifkan kembali.
4. Publish training.
5. Archive training.
6. Reset password.
7. Submit test.
8. Submit nilai essay.
9. Export jika perlu konfirmasi.

### Aturan Modal

1. Tidak menggunakan `alert()`, `confirm()`, atau `prompt()`.
2. Modal dibuat custom menggunakan Blade Component + Alpine.js.
3. Modal harus punya title, deskripsi, tombol batal, dan tombol konfirmasi.
4. Aksi danger harus memakai tombol merah.
5. Modal bisa ditutup dengan tombol close dan tombol batal.
6. Modal tidak boleh menutup otomatis saat proses loading belum selesai.

### Contoh

```text
Delete Permanen Karyawan

Data yang dihapus permanen tidak dapat dikembalikan.
Apakah Anda yakin ingin menghapus data ini?

[Batal] [Delete Permanen]
```

1. Page header.
2. Statistic card.

7. Status training terbaru.

## 5.10.3 Detail Training Karyawan

### Pola UI

Progress step + tablist.

### Komponen

1. Page header.

3. Progress step.
4. Summary card.
5. Tablist.
6. Material card.
7. Pre-test card.
8. Post-test card.
9. Result card.
10. Locked state.

### Progress Step

```text
[Pre-Test] → [Materi] → [Post-Test] → [Hasil]
```

### Tab

1. Ringkasan.
2. Materi.
3. Pre-Test.
4. Post-Test.
5. Hasil.

### Wireframe

```text
+--------------------------------------------------+
| Detail Training                         [Status] |

+--------------------------------------------------+

[✓ Pre-Test] → [● Materi] → [Locked Post-Test] → [Locked Hasil]

+--------------------------------------------------+
| Konten sesuai tab aktif                          |
+--------------------------------------------------+
```

### Locked State

| Kondisi              | Pesan                                                        |
| -------------------- | ------------------------------------------------------------ |

## 5.10.6 Mengerjakan Post-Test

### Pola UI

Sama seperti pre-test.

### Perbedaan

1. Post-test hanya terbuka setelah materi aktif sudah dibuka.
2. Jika post-test gagal dan retake masih tersedia, tampilkan tombol ulangi post-test.
3. Attempt keberapa ditampilkan di header.

### Header

```text
Post-Test: Judul Training
Percobaan ke-1 dari 3
```

### Error State

| Kondisi              | Pesan                                                        |
| -------------------- | ------------------------------------------------------------ |
| Materi belum lengkap | Buka semua materi terlebih dahulu sebelum mengerjakan post-test |
| Kesempatan habis     | Kesempatan mengulang post-test sudah habis                   |
| Submit gagal         | Jawaban post-test gagal dikirim                              |
| Essay menunggu nilai | Jawaban essay sedang menunggu penilaian admin                |

# 5.11 Responsive Behavior

------

## 5.11.1 Desktop

Breakpoint: `lg` ke atas.

Aturan:

1. Sidebar tampil penuh.
2. Topbar selalu tampil.
3. Table menggunakan layout normal.
4. Filter tampil horizontal di atas table.
5. Chart bisa 2 kolom.
6. Detail training bisa menggunakan 2 kolom.
7. Halaman test menggunakan sidebar nomor soal di kanan.

------

## 5.11.2 Tablet

Breakpoint: `md`.

Aturan:

1. Sidebar bisa collapse.
2. Content padding lebih kecil.
3. Filter bisa wrap ke baris baru.
4. Table dapat horizontal scroll.
5. Tablist dapat horizontal scroll.
6. Card dashboard 2 kolom.

------

## 5.11.3 Mobile

Breakpoint: `sm` ke bawah.

Aturan:

1. Sidebar menjadi drawer.
2. Topbar menampilkan hamburger menu.
3. Table kompleks dapat horizontal scroll.
4. Halaman Karyawan memakai card/list.
5. Filter bisa tetap di atas halaman, tetapi disusun vertikal.
6. Tablist horizontal scroll.
7. Nomor soal test menjadi sticky horizontal chips.
8. Button utama dapat full width.
9. Modal menggunakan lebar hampir penuh.

# 5.16 Accessibility Notes

Aturan aksesibilitas:

1. Semua button harus memiliki label yang jelas.
2. Icon button wajib memiliki tooltip dan `aria-label`.
3. Warna status tidak boleh menjadi satu-satunya pembeda; harus ada teks badge.
4. Input error harus tampil di bawah field.
5. Modal harus bisa ditutup dengan tombol close.
6. Focus state harus terlihat.
7. Tablist harus bisa digunakan dengan keyboard.
8. Kontras teks harus cukup pada background putih dan biru.
9. Loading tidak boleh menghilangkan konteks halaman.
10. Empty state harus menjelaskan tindakan berikutnya jika ada.
