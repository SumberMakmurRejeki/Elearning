# 02. Functional Requirements Document

# Sistem E-Learning Training Karyawan

------

## 2.1 Ringkasan Dokumen

Dokumen Functional Requirements Document atau FRD ini menjelaskan kebutuhan fungsional sistem E-Learning Training Karyawan secara lebih detail berdasarkan Product Requirements Document atau PRD.

Dokumen ini digunakan sebagai acuan untuk proses desain sistem, pembuatan database, pembuatan fitur, validasi form, pengaturan alur user, aturan bisnis, dan penanganan error state.

Sistem E-Learning Training Karyawan adalah aplikasi berbasis web yang digunakan untuk mengelola proses training internal perusahaan secara digital. Sistem ini memiliki dua role utama, yaitu Admin dan Karyawan.

Admin bertugas mengelola data master, training, materi, soal test, penugasan training, penilaian essay, monitoring progress, laporan, dan export data. Karyawan bertugas mengikuti training yang diberikan, mengerjakan pre-test, mengakses materi, mengerjakan post-test, dan melihat hasil training sesuai pengaturan admin.

------

## 2.2 Tujuan Functional Specification

Tujuan dokumen ini adalah:

1. Menjelaskan detail setiap fitur sistem.
2. Menjelaskan alur user untuk Admin dan Karyawan.
3. Menjelaskan validasi form pada setiap fitur utama.
4. Menjelaskan aturan bisnis sistem.
5. Menjelaskan error state yang mungkin terjadi.
6. Menjadi acuan developer dalam membangun sistem.
7. Menjadi acuan pengujian fitur saat sistem dikembangkan.

------

## 2.3 Aktor Sistem

### 2.3.1 Admin

Admin adalah pengguna yang memiliki akses untuk mengelola seluruh data dan proses training.

Admin dapat:

1. Login ke dashboard admin.
2. Mengelola data karyawan.
3. Mengelola data divisi.
4. Mengelola data jabatan.
5. Mengelola data training.
6. Mengelola materi training.
7. Mengelola soal pre-test dan post-test.
8. Menugaskan training ke karyawan, divisi, atau jabatan.
9. Melihat progress training karyawan.
10. Menilai jawaban essay.
11. Melihat hasil test.
12. Melihat laporan.
13. Mengekspor laporan ke PDF dan Excel.
14. Mengelola profil dan password admin.

### 2.3.2 Karyawan

Karyawan adalah pengguna yang mengikuti training yang sudah diberikan oleh admin.

Karyawan dapat:

1. Login menggunakan username dan password.
2. Melihat dashboard karyawan.
3. Melihat daftar training yang diberikan.
4. Melihat detail training.
5. Mengerjakan pre-test jika tersedia.
6. Mengakses materi training.
7. Mendownload materi training.
8. Mengerjakan post-test jika tersedia.
9. Melihat status training.
10. Melihat nilai jika diizinkan oleh admin.
11. Melihat riwayat training.
12. Mengubah password akun.

# 2.5 Alur Utama Sistem

## 2.5.1 Alur Utama Admin

1. Admin login ke sistem.
2. Sistem memvalidasi username dan password admin.
3. Admin masuk ke dashboard admin.
4. Admin mengelola master data seperti karyawan, divisi, dan jabatan.
5. Admin membuat data training.
6. Admin menambahkan materi training.
7. Admin membuat soal pre-test dan/atau post-test.
8. Admin mengatur bobot nilai soal.
9. Admin mengatur passing grade.
10. Admin mengatur apakah post-test dapat diulang.
11. Admin menugaskan training ke karyawan, divisi, atau jabatan.
12. Karyawan mengikuti training.
13. Admin memantau progress training karyawan.
14. Jika ada soal essay, admin menilai jawaban essay.
15. Sistem menghitung nilai akhir.
16. Admin melihat hasil test dan laporan.
17. Admin melakukan export laporan ke PDF atau Excel jika diperlukan.

## 2.5.2 Alur Utama Karyawan

1. Karyawan login ke sistem.
2. Sistem memvalidasi username dan password karyawan.
3. Karyawan masuk ke dashboard karyawan.
4. Karyawan melihat daftar training yang diberikan.
5. Karyawan membuka detail training.
6. Jika training menggunakan pre-test, karyawan wajib mengerjakan pre-test terlebih dahulu.
7. Setelah pre-test selesai, materi training dapat diakses.
8. Jika training tidak menggunakan pre-test, materi dapat langsung diakses.
9. Karyawan membuka dan/atau mendownload materi.
10. Sistem mencatat materi yang pernah dibuka oleh karyawan.
11. Jika training menggunakan post-test, karyawan mengerjakan post-test setelah materi diakses.
12. Sistem menyimpan jawaban karyawan.
13. Sistem menilai otomatis soal pilihan ganda.
14. Jika test memiliki essay, status menjadi “Menunggu Penilaian” sampai admin memberikan nilai.
15. Setelah nilai akhir tersedia, sistem menentukan status lulus atau tidak lulus.
16. Karyawan melihat status training.
17. Jika admin mengaktifkan pengaturan tampilkan nilai, karyawan dapat melihat nilai.

## 2.6.1 Dashboard Admin

### Deskripsi

Dashboard Admin digunakan untuk menampilkan ringkasan kondisi training secara cepat dan ringkas. Dashboard membantu admin memantau jumlah karyawan, jumlah training, progress training, nilai rata-rata, dan status kelulusan.

### Spesifikasi Fitur

| Item       | Detail                              |
| ---------- | ----------------------------------- |
| Nama fitur | Dashboard Admin                     |
| Aktor      | Admin                               |
| Menu       | Dashboard                           |
| Tujuan     | Menampilkan ringkasan data training |
| Input      | Filter bulan, tahun, training       |
| Output     | Card ringkasan dan grafik           |
| Hak akses  | Admin                               |

### Data yang Ditampilkan

1. Total karyawan.
2. Total training.
3. Training aktif.
4. Training selesai.
5. Rata-rata nilai post-test.
6. Jumlah karyawan lulus.
7. Jumlah karyawan tidak lulus.
8. Grafik progress training.
9. Grafik perbandingan nilai pre-test dan post-test.
10. Grafik kelulusan per training atau divisi.

### Alur Fitur

1. Admin membuka halaman Dashboard.
2. Sistem menampilkan ringkasan data training.
3. Admin dapat memilih filter bulan.
4. Admin dapat memilih filter tahun.
5. Admin dapat memilih filter training.
6. Sistem memperbarui data dashboard sesuai filter.
7. Sistem menampilkan grafik sesuai data yang tersedia.

### Aturan Bisnis

1. Dashboard hanya dapat diakses oleh Admin.
2. Data dashboard mengikuti filter yang dipilih.
3. Jika filter tidak dipilih, sistem menampilkan data bulan dan tahun berjalan.
4. Grafik hanya menampilkan data yang tersedia.
5. Dashboard dibuat ringkas dan tidak terlalu banyak elemen.

### Error State

| Kondisi                        | Pesan                                     |
| ------------------------------ | ----------------------------------------- |
| Data tidak tersedia            | Data dashboard belum tersedia             |
| Filter tidak menghasilkan data | Tidak ada data sesuai filter yang dipilih |
| Gagal memuat grafik            | Grafik gagal dimuat                       |

Fitur Daftar Training digunakan admin untuk membuat dan mengatur training. Training dapat menggunakan pre-test, post-test, keduanya, atau tanpa test sesuai pengaturan admin.

### Spesifikasi Fitur

| Nama fitur | Daftar Training                                              |
| Aktor      | Admin                                                        |
| Menu       | Master Training > Daftar Training                            |

| Input      | Judul, deskripsi, tanggal mulai, tanggal selesai, status, pengaturan test |
| Output     | Data training tersimpan                                      |

### Fitur

1. Tambah training.
2. Edit training.
3. Detail training.
4. Publish training.
5. Archive training.
6. Delete permanen training.
7. Search training.
8. Filter status training.
9. Filter bulan dan tahun.
10. Atur penggunaan pre-test.
11. Atur penggunaan post-test.
12. Atur nilai minimal kelulusan.
13. Atur pengulangan post-test.
14. Atur apakah nilai ditampilkan ke karyawan.

### Data Form Training

| Field                       | Keterangan                      |
| --------------------------- | ------------------------------- |
| Judul training              | Nama training                   |
| Deskripsi training          | Penjelasan training             |
| Tanggal mulai               | Tanggal training mulai tersedia |
| Tanggal selesai             | Tanggal akhir training          |
| Status training             | Draft, published, archived      |
| Menggunakan pre-test        | Ya/tidak                        |
| Menggunakan post-test       | Ya/tidak                        |
| Nilai minimal kelulusan     | Passing grade                   |
| Pengulangan post-test       | Tidak boleh/boleh               |
| Jumlah maksimal pengulangan | Jumlah maksimal percobaan       |
| Tampilkan nilai ke karyawan | Ya/tidak                        |

### Alur Tambah Training

1. Admin membuka menu Master Training > Daftar Training.
2. Admin klik Tambah Training.
3. Sistem menampilkan form tambah training.
4. Admin mengisi judul training.
5. Admin mengisi deskripsi training.
6. Admin mengisi tanggal mulai dan tanggal selesai.
7. Admin memilih penggunaan pre-test.
8. Admin memilih penggunaan post-test.
9. Jika post-test aktif, admin mengisi nilai minimal kelulusan.
10. Admin memilih pengaturan pengulangan post-test.
11. Jika pengulangan post-test diizinkan, admin mengisi jumlah maksimal pengulangan.
12. Admin memilih apakah nilai ditampilkan ke karyawan.
13. Admin memilih status training.
14. Admin klik Simpan.
15. Sistem memvalidasi data.
16. Sistem menyimpan data training.
17. Sistem menampilkan notifikasi berhasil.

### Alur Publish Training

1. Admin membuka detail training.
2. Admin memastikan training memiliki data yang diperlukan.
3. Admin klik Publish.
4. Sistem menampilkan konfirmasi.
5. Admin menyetujui konfirmasi.
6. Sistem mengubah status training menjadi published.
7. Training dapat diberikan kepada karyawan melalui penugasan.

### Alur Archive Training

1. Admin membuka detail training.
2. Admin klik Archive.
3. Sistem menampilkan konfirmasi.
4. Admin menyetujui konfirmasi.
5. Sistem mengubah status training menjadi archived.
6. Training yang diarsipkan tidak tersedia untuk penugasan baru.

### Validasi Form

| Field                 | Validasi                                |
| --------------------- | --------------------------------------- |
| Judul training        | Wajib, maksimal 150 karakter            |
| Deskripsi             | Opsional                                |
| Tanggal mulai         | Wajib                                   |
| Tanggal selesai       | Wajib                                   |
| Status                | Draft/published/archived                |
| Pre-test              | Wajib, ya/tidak                         |
| Post-test             | Wajib, ya/tidak                         |
| Passing grade         | Wajib jika post-test aktif, angka 0-100 |
| Pengulangan post-test | Wajib, tidak boleh/boleh                |
| Maksimal pengulangan  | Wajib jika pengulangan boleh, minimal 1 |
| Tampilkan nilai       | Wajib, ya/tidak                         |

### Aturan Bisnis

1. Training baru dapat disimpan sebagai draft.
2. Training dengan status published dapat ditugaskan ke karyawan, divisi, atau jabatan.
3. Training dengan status archived tidak dapat ditugaskan lagi.
4. Jika training menggunakan pre-test, karyawan wajib menyelesaikan pre-test sebelum membuka materi.
5. Jika training tidak menggunakan pre-test, materi dapat langsung diakses.
6. Jika training menggunakan post-test, karyawan dapat mengerjakan post-test setelah materi diakses.
7. Jika training tidak menggunakan post-test, training dapat selesai setelah materi diakses.
8. Passing grade digunakan untuk menentukan kelulusan berdasarkan nilai post-test.
9. Nilai hanya ditampilkan ke karyawan jika pengaturan tampilkan nilai diaktifkan.
10. Admin dapat melakukan delete permanen training.
11. Sebelum delete permanen, sistem wajib menampilkan konfirmasi.

### Error State

| Kondisi                               | Pesan                                             |
| ------------------------------------- | ------------------------------------------------- |
| Judul training kosong                 | Judul training wajib diisi                        |
| Tanggal mulai kosong                  | Tanggal mulai wajib diisi                         |
| Tanggal selesai kosong                | Tanggal selesai wajib diisi                       |
| Tanggal selesai sebelum tanggal mulai | Tanggal selesai tidak boleh sebelum tanggal mulai |
| Passing grade kosong                  | Passing grade wajib diisi jika post-test aktif    |
| Passing grade tidak valid             | Passing grade harus bernilai 0 sampai 100         |
| Maksimal pengulangan kosong           | Jumlah pengulangan wajib diisi                    |
| Training belum dipublish              | Training belum tersedia                           |
| Training archived                     | Training sudah diarsipkan                         |

## 2.11.6 Mengerjakan Post-Test

### Deskripsi

Fitur Mengerjakan Post-Test digunakan karyawan untuk mengerjakan post-test jika training menggunakan post-test. Kelulusan training ditentukan dari akses materi dan nilai post-test.

### Spesifikasi Fitur

| Item       | Detail                                                       |
| ---------- | ------------------------------------------------------------ |
| Nama fitur | Mengerjakan Post-Test                                        |
| Aktor      | Karyawan                                                     |
| Menu       | Training > Detail Training > Post-Test                       |
| Tujuan     | Mengerjakan post-test                                        |
| Input      | Jawaban pilihan ganda dan/atau essay                         |
| Output     | Jawaban tersimpan, nilai dihitung, status kelulusan ditentukan |
| Hak akses  | Karyawan yang ditugaskan                                     |

### Alur

1. Karyawan membuka detail training.
2. Karyawan memastikan materi sudah diakses.
3. Karyawan klik Kerjakan Post-Test.
4. Sistem memeriksa apakah post-test tersedia.
5. Sistem memeriksa apakah materi sudah diakses.
6. Sistem memeriksa batas pengulangan post-test.
7. Sistem menampilkan soal post-test aktif.
8. Karyawan menjawab semua soal.
9. Karyawan klik Submit.
10. Sistem memvalidasi semua soal sudah dijawab.
11. Sistem menyimpan jawaban.
12. Sistem menilai otomatis soal pilihan ganda.
13. Jika ada essay, status menjadi Menunggu Penilaian.
14. Jika tidak ada essay, sistem menghitung nilai akhir.
15. Sistem menentukan status lulus atau tidak lulus.
16. Jika nilai ditampilkan, karyawan dapat melihat nilai.
17. Jika nilai tidak ditampilkan, karyawan hanya melihat status.

### Validasi Submit

| Kondisi       | Validasi                             |
| ------------- | ------------------------------------ |
| Pilihan ganda | Wajib memilih salah satu jawaban     |
| Essay         | Wajib diisi                          |
| Semua soal    | Semua soal wajib dijawab             |
| Pengulangan   | Tidak boleh melebihi batas percobaan |

### Aturan Bisnis

1. Post-test hanya muncul jika training menggunakan post-test.
2. Post-test hanya dapat dikerjakan setelah materi diakses.
3. Semua soal wajib dijawab.
4. Soal pilihan ganda dinilai otomatis.
5. Soal essay menunggu penilaian admin.
6. Jika test memiliki essay, status menjadi “Menunggu Penilaian” sampai admin memberikan nilai.
7. Nilai akhir dihitung dengan normalisasi ke skala 100.
8. Rumus nilai akhir adalah total nilai diperoleh dibagi total bobot soal dikali 100.
9. Karyawan dinyatakan lulus jika sudah mengakses materi dan nilai post-test lebih besar atau sama dengan passing grade.
10. Jika nilai post-test lebih kecil dari passing grade, karyawan tidak lulus.
11. Jika admin mengizinkan pengulangan, karyawan dapat mengulang post-test sesuai batas yang ditentukan.
12. Jika batas pengulangan habis, karyawan tidak dapat mengerjakan ulang post-test.
13. Nilai hanya ditampilkan jika admin mengaktifkan pengaturan tampilkan nilai.

### Error State

| Kondisi                  | Pesan                                          |
| ------------------------ | ---------------------------------------------- |
| Materi belum diakses     | Akses materi terlebih dahulu sebelum post-test |
| Post-test tidak tersedia | Post-test tidak tersedia untuk training ini    |
| Soal belum dijawab       | Masih ada soal yang belum dijawab              |
| Jawaban essay kosong     | Jawaban essay wajib diisi                      |

# 2.14 Aturan Bisnis Global

## 2.14.1 Aturan Login

1. User login menggunakan username dan password.
2. Sistem memeriksa role user.
3. Admin diarahkan ke dashboard admin.
4. Karyawan diarahkan ke dashboard karyawan.
5. User nonaktif tidak dapat login.
6. Password disimpan dalam bentuk terenkripsi.

## 2.14.2 Aturan Status Data

1. Data aktif dapat digunakan untuk transaksi baru.
2. Data nonaktif tidak dapat dipilih untuk transaksi baru.
3. Data nonaktif tetap tampil pada riwayat.
4. Admin dapat mengaktifkan kembali data nonaktif.
5. Admin dapat melakukan delete permanen data.
6. Sebelum delete permanen, sistem wajib menampilkan konfirmasi.
7. Data yang sudah dihapus permanen tidak dapat dikembalikan.

## 2.14.3 Aturan Training

1. Training dapat berstatus draft, published, atau archived.
2. Training draft belum dapat ditugaskan.
3. Training published dapat ditugaskan.
4. Training archived tidak dapat ditugaskan lagi.
5. Training dapat menggunakan pre-test, post-test, keduanya, atau tanpa test.
6. Training dapat memiliki lebih dari satu materi.
7. Training dapat ditugaskan ke karyawan, divisi, atau jabatan.

## 2.14.4 Aturan Akses Training

1. Karyawan hanya dapat mengakses training yang ditugaskan kepadanya.
2. Jika training menggunakan pre-test, karyawan wajib menyelesaikan pre-test terlebih dahulu.
3. Jika training tidak menggunakan pre-test, materi dapat langsung diakses.
4. Jika training menggunakan post-test, post-test dapat dikerjakan setelah materi diakses.
5. Jika training tidak menggunakan post-test, training selesai setelah materi diakses.

## 2.14.5 Aturan Materi

1. Satu training dapat memiliki lebih dari satu materi.
2. Materi dapat berupa file atau link.
3. Materi aktif tampil untuk karyawan.
4. Materi nonaktif tidak tampil untuk karyawan.
5. Sistem mencatat materi yang pernah dibuka.
6. Sistem tidak mencatat persentase tontonan video.

## 2.14.6 Aturan Test

1. Soal test terdiri dari pre-test dan post-test.
2. Pre-test dan post-test dikelola dalam satu modul soal.
3. Satu test dapat berisi campuran soal pilihan ganda dan essay.
4. Semua soal wajib dijawab sebelum submit.
5. Pilihan ganda dinilai otomatis.
6. Essay dinilai manual oleh admin.
7. Jika test memiliki essay, status menjadi Menunggu Penilaian sampai admin memberikan nilai.
8. Bobot soal ditentukan oleh admin.
9. Nilai akhir dihitung dengan normalisasi ke skala 100.

## 2.14.7 Aturan Nilai

1. Nilai pilihan ganda dihitung otomatis oleh sistem.
2. Nilai essay diinput oleh admin.
3. Nilai essay tidak boleh melebihi bobot maksimal soal.
4. Nilai akhir dihitung setelah semua nilai tersedia.
5. Rumus nilai akhir adalah:

Nilai akhir = total nilai diperoleh / total bobot soal x 100

1. Nilai akhir post-test digunakan untuk menentukan kelulusan.
2. Nilai hanya ditampilkan ke karyawan jika admin mengaktifkan pengaturan tampilkan nilai.

## 2.14.8 Aturan Kelulusan

1. Kelulusan ditentukan dari akses materi dan nilai post-test.
2. Karyawan lulus jika sudah mengakses materi dan nilai post-test lebih besar atau sama dengan passing grade.
3. Karyawan tidak lulus jika nilai post-test lebih kecil dari passing grade.
4. Jika post-test memiliki essay, status kelulusan belum final sampai essay dinilai.
5. Jika training tidak menggunakan post-test, status training dapat selesai setelah materi diakses.

## 2.14.9 Aturan Pengulangan Post-Test

1. Admin dapat menentukan apakah post-test boleh diulang.
2. Jika tidak boleh mengulang, karyawan hanya dapat mengerjakan post-test satu kali.
3. Jika boleh mengulang, admin wajib mengisi jumlah maksimal pengulangan.
4. Karyawan tidak dapat mengulang post-test setelah batas percobaan habis.
5. Sistem mencatat percobaan post-test keberapa.

## 2.14.10 Aturan Penugasan

1. Training dapat ditugaskan ke karyawan, divisi, atau jabatan.
2. Sistem membuat progress training untuk setiap karyawan yang menerima penugasan.
3. Sistem mencegah duplikasi training untuk karyawan yang sama.
4. Satu karyawan hanya boleh memiliki satu progress aktif untuk satu training yang sama.
5. Karyawan nonaktif tidak menerima penugasan baru.

## 2.14.11 Aturan Export

1. Export tersedia dalam format PDF dan Excel.
2. Export dilakukan berdasarkan halaman laporan.
3. Export mengikuti filter yang dipilih.
4. Jika data laporan kosong, sistem tidak membuat file export.
5. Sistem menampilkan pesan jika data laporan kosong.
