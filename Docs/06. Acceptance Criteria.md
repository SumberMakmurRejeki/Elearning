# 06. Acceptance Criteria

# Sistem E-Learning Training Karyawan

------

## 6.1 Ringkasan Dokumen

Dokumen Acceptance Criteria ini menjelaskan kriteria penerimaan fitur pada Sistem E-Learning Training Karyawan.

Acceptance Criteria digunakan sebagai acuan untuk:

1. Memastikan setiap fitur berjalan sesuai kebutuhan.
2. Membantu developer memahami hasil akhir yang harus dicapai.
3. Membantu proses testing fitur.
4. Menjadi dasar validasi apakah fitur sudah selesai atau belum.
5. Menjaga agar implementasi sesuai dengan PRD, FRD, User Flow, ERD, dan UI Spec.

Dokumen ini menggunakan format sederhana:

```text
Given kondisi awal
When aksi dilakukan
Then hasil yang diharapkan terjadi
```

------

# 6.3 Authentication & Role Access

------

## AC-AUTH-001 — Login Admin Berhasil

Given admin memiliki akun aktif
When admin login menggunakan username dan password yang valid
Then sistem mengarahkan admin ke dashboard admin

------

## AC-AUTH-002 — Login Karyawan Berhasil

Given karyawan memiliki akun aktif
When karyawan login menggunakan username dan password yang valid
Then sistem mengarahkan karyawan ke dashboard karyawan

------

## AC-AUTH-003 — Login Gagal Karena Username atau Password Salah

Given user berada di halaman login
When user memasukkan username atau password yang salah
Then sistem menampilkan pesan login gagal

------

## AC-AUTH-004 — Login Gagal Karena Akun Nonaktif

Given user memiliki akun nonaktif
When user mencoba login ke sistem
Then sistem menolak login dan menampilkan pesan bahwa akun sedang nonaktif

------

## AC-AUTH-005 — Akses Admin Ditolak untuk Karyawan

Given karyawan sudah login
When karyawan mencoba membuka halaman dashboard admin
Then sistem menolak akses dan mengarahkan karyawan ke halaman yang sesuai

------

## AC-AUTH-006 — Akses Karyawan Ditolak untuk Admin

Given admin sudah login
When admin mencoba membuka halaman khusus karyawan melalui URL langsung
Then sistem menolak akses dan mengarahkan admin ke dashboard admin

------

# 6.4 Dashboard Admin

------

## AC-DASH-001 — Dashboard Admin Tampil

Given admin sudah login
When admin membuka halaman dashboard admin
Then sistem menampilkan ringkasan data training

------

## AC-DASH-002 — Card Ringkasan Dashboard Tampil

Given data training tersedia
When dashboard admin dimuat
Then sistem menampilkan total karyawan, total training, training aktif, training selesai, rata-rata nilai post-test, jumlah lulus, dan jumlah tidak lulus

------

## AC-DASH-003 — Filter Dashboard Berjalan

Given admin berada di dashboard admin
When admin memilih filter bulan, tahun, atau training
Then sistem memperbarui data dashboard sesuai filter yang dipilih

------

## AC-DASH-004 — Grafik Dashboard Tampil

Given data dashboard tersedia
When admin membuka dashboard
Then sistem menampilkan grafik progress training, grafik perbandingan nilai pre-test dan post-test, dan grafik kelulusan

------

## AC-DASH-005 — Empty State Dashboard

Given tidak ada data sesuai filter yang dipilih
When dashboard dimuat
Then sistem menampilkan empty state bahwa data dashboard belum tersedia

------

## AC-DASH-006 — Error State Dashboard

Given sistem gagal memuat data dashboard
When admin membuka dashboard
Then sistem menampilkan pesan bahwa data dashboard gagal dimuat
